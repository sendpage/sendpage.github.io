• To view the SourceForge project, check the Sendpage Project Page
  • To download versions of sendpage, use sourceforge or use my local mirror.
  • To download versions of Device::SerialPort, use CPAN or use my local mirror.
  • Searchable database list of known TAP Paging Centrals world-wide.
  • If you have problems or need help with Sendpage, see if the sendpage-devel mailing list archive can help you. I'm usually quick to answer any questions.
  • To read the TAP v1.8 specification, read either
    • the PCIA original
    • the Motorola mirror
  • To read the SNPP specification, see RFC 1861
  • To punish yourself, read the old sendpage web site
  • If Sendpage isn't for you, check out some of the other gratis paging software solutions:
    • QPage
    • Beepage
  • If you have any other questions regarding Sendpage, please drop me an email at sendpage@outflux.net.
  • Companies and individuals that have told me they're using Sendpage:
    • Counterpoint Networking (a US ISP) uses it for network/system alerting, and as an email-to-pager gateway for employees.
    • A big (unnamed) US energy company uses it for exhaust level alerting.
    • Business Serve plc (a UK ISP) uses it with Nagios for network/system alerting, and as an email-to-pager gateway for employees.
    • A big (unnamed) tobacco company uses it for network/system alerting.
    • OSDL, home to Linus Torvalds, uses it with Nagios for network/system alerting. (I work here, too.)
